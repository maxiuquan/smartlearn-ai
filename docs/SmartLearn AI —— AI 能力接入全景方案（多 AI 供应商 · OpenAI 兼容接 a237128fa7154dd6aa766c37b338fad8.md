## 三、核心学习场景

### 3.1 AI 智能答疑(RAG · 核心)

- **用途**:学生就任意知识点 / 题目提问,AI 依据检索到的本站知识点作答,并给出引用。
- **接口**:Embeddings 向量化问题 → Milvus 召回 → Chat Completions 流式生成 → Moderation 审核。
- **要点**:系统提示强制「只依据检索内容作答,标注引用 kp-id,无依据则说明不臆造」;SSE 流式输出提升体验。
- **注意**:每次调用落 token 成本;低置信度提示「建议教研复核」。

### 3.2 题目解析与追问

- **用途**:对任一题目一键生成分步解析,并支持「没懂,换种讲法 / 再详细点」的追问。
- **接口**:Chat Completions(数学题配 LaTeX 输出规范)。
- **要点**:解析须与题库 answer 对齐;生成结果可缓存到 `ai_conversations`,重复题目复用。

### 3.3 错题诊断

- **用途**:分析学生错因(概念混淆 / 计算失误 / 审题错误),定位对应薄弱知识点,给出针对性建议。
- **接口**:Chat Completions,输入=题目+正确解+学生作答。
- **要点**:输出结构化错因标签,回写错题本,驱动「错题 Boss 战」。

### 3.4 个性化学习路径

- **用途**:结合 `user_word_progress` / 作答记录 + `math.json` 依赖图,推荐下一步该学 / 该复习什么。
- **接口**:Embeddings(相似薄弱点聚类)+ Chat(生成可读的学习计划)。
- **要点**:AI 负责「解释与编排」,掌握度与到期调度仍由 SRS 算法计算,避免让 LLM 算数。

## 四、英语模块 AI 接入

### 4.1 长难句拆解(游戏 5)

- 生成句子成分标注(主谓宾、定状补、从句)作为「长难句拆解」游戏的答案数据。**接口**:Chat。**要点**:批量离线预生成 + 人工抽检,避免运行时实时调用。

### 4.2 作文 / 翻译批改

- 对英语写作 / 翻译按考研评分维度打分并给出修改建议。**接口**:Chat(建议 gpt-4o 提升批改质量)。**要点**:给出分项得分 + 逐句批注 + 范文改写。

### 4.3 例句 / 助记 / 同反义生成

- 为词库补齐例句、联想助记、同义 / 反义词,喂给「闪卡速记」「同反义连线」「语境速填」等游戏。**接口**:Chat + Embeddings(相似度筛选干扰项)。**要点**:离线批量生成入库,人工复核后固化,避免每局实时调用。

### 4.4 发音与听力(TTS)

- 为「拼写蜂」「听音辨词」生成单词 / 例句音频。**接口**:TTS。**要点**:音频**预生成并存 MinIO/CDN**,按 `word_id` 缓存,不在游戏内实时合成以控成本与延迟。

### 4.5 口语跟读与听写(Whisper)

- 学生朗读单词 / 句子,识别后比对评分;或听写练习。**接口**:Whisper(STT)+ Chat(评分反馈)。**优先级** P2。

## 五、数学模块 AI 接入

### 5.1 解题步骤讲解

- 为「极限速算」「解题闯关地图」生成分步讲解与提示。**接口**:Chat。**要点**:LaTeX 统一 `$...$`,前端 KaTeX/MathJax 渲染。

### 5.2 证明拆分与校验(游戏 19)

- 把证明题拆成有序步骤供「证明步骤排序」使用,并校验逻辑顺序。**接口**:Chat(建议 gpt-4o)。**要点**:生成结果必须人工复核,数学正确性 LLM 不完全可靠。

### 5.3 公式讲解与相似题生成

- 生成「公式连连看」的公式—适用条件配对数据;为薄弱点生成同类相似题。**接口**:Chat。**要点**:相似题须标注知识点并入 P0 数据校验,防孤立引用。

## 七、内容生产与数据治理 AI 接入

### 7.1 题库 / 词表生成辅助(人工复核闭环)

- 用 AI 草拟题目、选项、解析、词条释义,**必须经教研人工复核后入库**。**接口**:Chat。**注意**:生成内容标识 + 版权合规;AI 生成≠可直接商用。

### 7.2 知识点自动标注

- 为题目自动打 `knowledge_points` 标签、估计难度。**接口**:Embeddings(匹配知识点)+ Chat(校验)。

### 7.3 数据清洗(修复现存 bug)

- 批量清洗 solution 中的口水话(如 `math-q016` 的「等等,让我重新检查」)、识别答案与解析矛盾(`math-q003`)、标记不可解题(`math-q014`)。**接口**:Chat。**注意**:AI 只做「发现与建议」,最终修正人工确认。

### 7.4 语义去重与相似题聚类

- 用 Embeddings 计算题目 / 词条相似度,去重、发现重复、组织「相似题」推荐。**接口**:Embeddings。

### 7.5 词库增强

- 补齐 `synonyms` / `antonyms` / 例句 / 词频 / 助记等字段(游戏依赖),统一入 `vocabulary_words`。**接口**:Chat + Embeddings。

## 八、平台底层 AI 能力

### 8.1 语义搜索

- 全站题目 / 知识点 / 词条的语义检索(超越关键词)。**接口**:Embeddings + Milvus。

### 8.2 内容安全(Moderation · P0)

- 对 **AI 每一次输出**与**所有 UGC**(昵称、聊天、组队)双向过审。**接口**:Moderation。**注意**:国内运营还需叠加中文合规词库 / 第三方审核,Moderation 作为第一道防线。

### 8.3 学习报告 / 周报生成

- 把作答、掌握度、复习数据总结成自然语言周报与激励文案。**接口**:Chat。**优先级** P2。

### 8.4 智能推荐与复习调度

- 结合到期队列与薄弱点,生成「今日建议」;调度由 SRS 算法负责,AI 负责解释与排版。**接口**:Chat + Embeddings。

## 九、AI 工程架构(基于标准 OpenAI 接口)

<aside>
🏗️

所有 AI 调用统一收口到 **ai-engine 内的「AI 网关 / 路由层」(8001)**:对外统一成标准 OpenAI 兼容形态,对内按「能力 × 科目」把请求路由到不同供应商,便于统一鉴权、配额、缓存、成本核算、护栏与热切换,避免各处散调。

</aside>

### 9.1 调用链

```jsx
Client → API(鉴权/配额/限流)
  → ai-engine → Embeddings 向量化 query
  → Milvus 召回 知识点/题目 → 组装带引用 Prompt
  → Chat Completions(流式) → Moderation 审核
  → SSE 回传 + 异步落库(问答/token成本/掌握度事件)
```

### 9.2 多 AI 供应商路由(数学 / 英语 / 图片分开)

<aside>
🔀

**按「能力 × 科目」路由到不同 AI**:各家在不同领域各有所长(数学推理、英语语言、图像生成),由 AI 网关按下表分发,并支持主 / 备切换、灰度与 A/B。

</aside>

| 场景 | 路由到(按你选型) | 能力 / 接口 | 要点 |
| --- | --- | --- | --- |
| 数学答疑 / 解题 / 证明 | 擅长推理的模型 | Chat | 强推理 + 严格 LaTeX;证明须人工复核 |
| 英语答疑 / 作文批改 / 长难句 | 擅长语言的模型 | Chat | 语言润色、批改、句法标注 |
| 图片生成(图词配对 / 配图) | 图像生成 AI | Image | 与文本模型解耦,独立选型 |
| 发音(TTS) | 语音合成供应商 | TTS | 可独立选型,预生成入库 |
| 口语听写(STT) | 语音识别供应商 | Whisper 类 | 可独立选型 |
| 向量检索(RAG) | **统一一家** embedding | Embeddings | ⚠️ 全局必须统一,不可混用(见下) |
| 内容安全 | 审核服务 | Moderation | 可叠加国内合规审核 |

**多供应商设计要点**

- **统一适配层(Adapter)**:优先选「OpenAI 兼容接口」的供应商,直接统一成同一调用形态;不兼容的写适配器转换。
- **路由配置化**:以 `(capability, subject)` 为路由键映射 `provider + model`,可热切换、灰度、A/B,不改业务代码。
- **⚠️ Embedding 必须全局统一**:同一 Milvus 集合的向量必须来自**同一 embedding 模型**;更换 embedding 供应商 = **全量重建索引**。检索用的 embedding 与生成答案用的 Chat 可以是不同家,互不影响。
- **每家独立管理**:各供应商独立 API Key、配额、限流与成本核算;调用记录统一落 `ai_conversations` 并标注 `provider` / `model`。
- **主 / 备降级**:每个场景配主备两家,超时 / 报错 / 超额自动切换或回退缓存,保证可用性。
- **分家评测与调优**:每场景维护评测集,按供应商对比**正确率 / 幻觉率 / 成本 / 延迟**,择优路由;Prompt 需针对各家分别调优并版本化。
- **同家内再分档**:选定供应商后,简单任务用小模型、难题 / 批改用大模型,进一步控成本。

### 9.2.1 你的可用模型落地映射

<aside>
🎯

把你现有的模型按「能力 × 科目」对号入座如下;标 ❗ 的是**当前缺口,上线前必须补齐**。

</aside>

| 场景 | 选用模型 | 定位 |
| --- | --- | --- |
| 在线主力:答疑 / 解析 / 诊断 / 标注 / 生成(默认) | GLM-4.7 Flash | 国内合规、便宜快,承接大部分在线量 |
| 最高端:数学难题 / 证明校验 / 作文精批 | DeepSeek V4 Flash | 强推理,难题与高准确度升级档 |
| 离线内容生产:题目 / 例句 / 长难句标注 / 范文 | Gemini 2.5 Pro Exp(free) | 质量高但限速,仅离线批量(见警示) |
| 图片生成:图词配对 / 配图 | CogView-3 Flash(主)+ 硅基流动 SD 小模型(备)+ Agnes(备) | 三图像源互为降级 |
| 多模态 / 视频生成(新能力) | Agnes(文本 / 图像 / 视频) | 备用文本源 + 视频生成(见下,P2/P3) |
| 发音 TTS / 听写 STT | 硅基流动 CosyVoice(TTS)/ SenseVoice(STT) | Agnes 不含音频,语音单独用硅基流动 |
| 向量检索 Embedding(RAG 基石) | 硅基流动 bge-m3(免费) | 全局统一;免费额度足够起步 |
| 内容安全 Moderation | ❗未提供 | 建议 GLM 初筛 + 国内第三方审核 |

**必须先解决的缺口与警示**

- **✅ Embedding 已定:硅基流动 bge-m3(免费)**。❗注意:「免费 9B 模型」是对话 LLM,**不能当 embedding 用**——向量检索必须用专门的 embedding 模型(bge-m3),且**全局统一**,中途更换需全量重建向量索引。免费 9B LLM 可作在线文本的廉价备用。
- **✅ Agnes = 多模态(文本 / 图像 / 视频)**:定位为**备用文本源 + 备用图像源 + 视频生成**。视频为新增能力,适用于:知识点讲解短视频、解题 / 证明动画演示、游戏过场 / 营销素材。因成本 / 延迟高,**走离线预生成 + 缓存**,列为 P2/P3。
- **⚠️ Gemini 2.5 Pro Exp(free)慎用在线**:免费实验版有严格速率 / 配额、稳定性无保障,且为境外模型,用于国内 C 端在线高并发存在**数据出境 + 生成式 AI 备案**合规风险。**建议只用于离线内容生产**,不进在线答疑主链路。
- **主备降级**:在线文本 GLM-4.7 Flash ↔ DeepSeek V4 Flash(↔ 硅基流动免费 9B 兵备);图片 CogView-3 Flash ↔ 硅基流动 SD ↔ Agnes 三备。
- **成本分档**:高并发日常走 GLM(便宜),仅难题 / 证明 / 精批升级到 DeepSeek,Gemini 只跑离线,整体成本可控。

### 9.3 成本控制与缓存

- Embedding 结果与高频问答 / 题目解析**缓存**复用;
- 音频、配图**预生成入库**,不实时调用;
- 按会员配额限流(Redis 计数),每次调用 token 成本落 `ai_conversations`;
- 内容生产走**批量离线**,不占线上额度。

### 9.4 Guardrails(护栏)

- 强制引用来源、无依据不臆造;
- 数学统一 LaTeX 规范;
- 输入输出双向 Moderation;
- 低置信度 → 「建议教研复核」,不直接给可能错误的答案。

### 9.5 Prompt 管理与评测

- Prompt 版本化管理;
- 维护评测集(query → 期望知识点 / 期望答案),回归监控**召回率、正确率、幻觉率**;
- 上线新 Prompt / 换模型前先跑评测。
