### 2.2 安全漏洞(上线前必须修复)

从 `docker-compose.yml` 与 `.env.example` 精读发现:

- [ ] &#x20;**JWT 密钥硬编码弱值**:`JWT_SECRET=${JWT_SECRET:-your-secret-key}`,默认值可被任何人伪造 token
- [ ] &#x20;**MinIO 凭据硬编码**:`MINIO_ACCESS_KEY=minioadmin` / `MINIO_SECRET_KEY=minioadmin`,且未走环境变量
- [ ] &#x20;**数据库弱口令**:Postgres 默认 `postgres/postgres`
- [ ] &#x20;**内部服务端口全部对宿主机暴露**:`5432`(PG)、`6379`(Redis)、`19530`(Milvus)、`9000/9091` 直接映射到宿主机,生产环境应仅在内网 `smartlearn-network` 暴露,不 `ports` 到主机
- [ ] &#x20;**Redis 无密码**:`REDIS_PASSWORD=` 为空,且端口外露
- [ ] &#x20;**默认环境为 production**:`.env.example` 中 `ENVIRONMENT=production`,极易误导直接带默认弱口令上生产
- [ ] &#x20;**无 TLS/HTTPS 证书**:nginx 挂载 `./ssl` 但仓库无证书,443 无法真正启用
- [ ] &#x20;**CORS 过宽风险**:需生产环境收敛到真实域名,禁止 [localhost](http://localhost)
- [ ] &#x20;**密钥管理**:应引入 Secrets 管理(Vault / 云 KMS / Docker secrets),禁止明文入库

### 2.3 数据质量 Bug(精读题库发现的具体问题)

- [ ] &#x20;**`math-q016`(格林公式)残留 AI 生成痕迹**:solution 中出现「等等,让我重新检查……」这类自我纠错口水话,且标题为「格林公式应用」但最终结果为 0(过于平凡),不适合作为示范题。**需清洗全部题目的 solution 文本**
- [ ] &#x20;**`math-q014`(条件极值)题目本身有问题**:solution 结尾承认「本题条件 xyz=1 与 x+y+z=1 在实数范围内可能无解,需进一步分析」——即答案与过程自相矛盾,是一道未完成/错误的题
- [ ] &#x20;**`math-q003`(间断点)答案与解析矛盾**:对 `x=kπ+π/2` 点,answer 标为「无穷间断点」,但 solution 判定为「可去间断点」
- [ ] &#x20;**知识点引用完整性存疑**:题库引用了 `kp-6-3-x`(曲线积分)、`kp-7-x`(无穷级数)、`kp-8-x`(微分方程)等知识点,需确认 `math.json` 是否真正包含第 6\~8 章完整节点,否则会产生**孤立外键引用**
- [ ] &#x20;**建立数据校验脚本**:自动检查①每道题的 `knowledge_points` 都能在知识点库中找到;②answer 与 solution 一致性(至少人工复核);③LaTeX 可正确渲染
- [ ] &#x20;**补齐空目录**:`data/exam-papers/`(历年真题)与 `data/exercise-books/`(习题册)当前为空

## 八、🎮 学习游戏化模块设计(共 25 款学习游戏,含 16 款英语单词游戏)

<aside>
🎯

本模块把「记忆 → 练习 → 复习」全流程游戏化,覆盖**英语单词/词汇、数学、跨科目社交**三大类,共 **25 款游戏(其中英语单词/词汇类 16 款)**。**所有英语单词游戏共用同一份账号级单词进度(统一掌握度与复习调度,详见 8.2)**。目标:提升日活与次日留存、延长单次学习时长、驱动付费转化;同时严守防沉迷与内容合规。

</aside>

### 8.1 游戏化基础框架(所有游戏共用)

- **积分 / 经验(XP)**:每局产出积分与 XP,积分用于商城,XP 用于升级
- **等级与段位**:XP 升级(学徒→学者→大师…),PVP 类另设段位赛季
- **成就徽章**:里程碑成就(连续打卡 30 天、攻克 1000 词、章节全通)
- **排行榜**:好友榜 / 全站日榜周榜 / 段位榜,支持匿名昵称
- **连续打卡 Streak**:连续学习天数,断签有「补签卡」(内购点)
- **虚拟货币与商城**:金币购买道具/皮肤/头像框,**仅站内使用、不可提现**
- **每日任务 / 赛季**:每日任务给新鲜感,赛季(2\~4 周)驱动长期回流

### 8.2 统一单词进度系统(所有单词游戏共享)

<aside>
🔗

**核心原则:一个账号,一份单词进度。** 全部 16 款英语单词游戏不各记一套,而是统一读写同一份「用户单词掌握库」(`user_word_progress`)。无论在哪个游戏里学到、答对或答错某个单词,它的掌握度、复习时间、对错次数都会实时同步到账号级进度,并反映到其他所有游戏与总进度条。

</aside>

- **单一数据源(Single Source of Truth)**:每个单词对每个用户只有一条进度记录,以 `(user_id, word_id)` 为唯一键;所有游戏通过统一的「单词进度服务(Vocabulary Progress Service)」读写,杜绝各游戏各记一套。
- **统一掌握度模型**:用同一套 SRS(间隔重复,SM-2 / 改进版)计算 `mastery_level`、`ease_factor`、`interval`、`next_review_at`,所有游戏共用,不因游戏不同而重复计算或互相冲突。
- **事件驱动同步**:每局结束后提交「单词事件」(答对 / 答错 / 跳过 / 超时 + 来源游戏 + 反应时长),由进度服务统一结算掌握度并回写下次复习时间。
- **全局进度可见且一致**:账号首页展示「已掌握 X / 目标词表 N」「今日待复习 M」,数值在所有游戏中完全一致;「记忆迷宫」「闪卡速记」等复习类游戏直接取用同一到期队列。
- **词表与 ID 统一**:以 `vocabulary/kaoyan-words.json`、`word-books.json` 的 `word_id` 为全局唯一标识,考研核心词、真题高频词等词表共享同一份进度,不重复计词。
- **跨端一致**:进度存于服务端,移动端 / Web / 小程序多端登录后完全同步。

统一进度数据模型(建议 `user_word_progress`):

```json
{
  "user_id": "u_10086",
  "word_id": "kw_abandon",
  "status": "review",
  "mastery_level": 3,
  "ease_factor": 2.5,
  "interval_days": 4,
  "next_review_at": "2026-07-07T00:00:00+08:00",
  "review_count": 6,
  "correct_count": 5,
  "wrong_count": 1,
  "last_source_game": "spelling-bee",
  "updated_at": "2026-07-03T15:00:00+08:00"
}
```

- **所有游戏统一提交的「单词事件」**:`user_id`、`word_id`、`game_id`、`result`(correct / wrong / skip / timeout)、`response_ms`、`occurred_at`,进度服务据此统一更新掌握度与复习计划。
- **落地要点**:纳入 P0 数据校验——任一游戏引用的 `word_id` 必须能在词库中找到,避免孤立引用。

### 8.4 逐款游戏详细设计

#### A. 英语单词 / 词汇类(16 款)

- <br />
  1. 词义消消乐 Word Match Blast
  - **玩法**:限时内把英文单词与中文释义(或近义/反义)配对消除,连击(combo)加分;随等级加入形近词、近义词干扰项
  - **学习目标**:快速建立词形—词义映射,高频扩充词汇量
  - **核心机制**:连击倍率、限时、道具(提示/冻结时间/炸弹)、难度梯度
  - **数据依赖**:`kaoyan-words.json`(词/释义/近反义)
  - **留存/商业价值**:轻量高频,碎片时间首选;道具为核心内购点
  - **技术要点**:纯前端可离线玩,成绩后同步;干扰项按词向量相似度生成
- <br />
  1. 拼写蜂 Spelling Bee
  - **玩法**:听发音或看释义,键盘拼出单词;拼错高亮并扣生命值,限次数
  - **学习目标**:巩固拼写与发音,专攻易错词
  - **核心机制**:生命值、错词自动入错词本、分级词库、连拼奖励
  - **数据依赖**:词库 + 音频(TTS 或预生成音频文件)
  - **留存/商业价值**:与错词本、复习闭环强联动
  - **技术要点**:TTS/预生成音频;用编辑距离判定近似拼写给出提示
- <br />
  1. 词根词缀树 Root & Affix Tree
  - **玩法**:给定词根/词缀推断或拼装派生词,按词根归类单词,逐步点亮「词根树」图鉴
  - **学习目标**:以构词法批量记忆,提升生词推断能力
  - **核心机制**:技能树式解锁、图鉴收集、成就
  - **数据依赖**:需**新增**词根词缀数据 `roots.json`(词根—词缀—派生词关系)
  - **留存/商业价值**:高差异化、深度付费内容
  - **技术要点**:词根-单词关系图数据建模
- <br />
  1. 语境速填 Cloze Sprint
  - **玩法**:真题/例句挖空,限时从选项选最合适的词(考辨析与搭配)
  - **学习目标**:语境用法、词义辨析、固定搭配
  - **核心机制**:连对加倍、限时、即时解析
  - **数据依赖**:句库(可源自真题,**注意版权授权**)
  - **留存/商业价值**:贴近考试,付费题库入口
  - **技术要点**:句子级数据 + 干扰项生成(近义/形近)
- <br />
  1. 长难句拆解 Sentence Untangle
  - **玩法**:拖拽标注句子成分(主谓宾、定状补、从句),逐步拆解考研长难句
  - **学习目标**:阅读理解与语法结构分析
  - **核心机制**:关卡制、逐步提示、结构评分
  - **数据依赖**:带句法结构标注的句库
  - **留存/商业价值**:阅读提分核心卖点,强差异化
  - **技术要点**:句法标注数据 + 拖拽交互;可用句法解析器预处理生成答案
- <br />
  1. 词汇 PK 擂台 Vocabulary Duel
  - **玩法**:实时或异步双人抢答词汇题,答对多者胜;支持匹配与好友对战
  - **学习目标**:词汇熟练度 + 竞技激励
  - **核心机制**:匹配、段位、赛季、观战
  - **数据依赖**:词库 + 题目
  - **留存/商业价值**:社交裂变、强留存、赛季通行证付费
  - **技术要点**:实时对战需 WebSocket + 匹配服务;异步模式用对手快照
- <br />
  1. 闪卡速记 Flashcard Rush
  - **玩法**:正面单词 / 背面释义的闪卡快速翻看,自评「记得 / 模糊 / 忘了」,系统据此安排下次复习
  - **学习目标**:高效自测、快速过词与查漏
  - **核心机制**:三档自评、SRS 到期队列、连续打卡
  - **数据依赖**:`kaoyan-words.json` + 统一 `user_word_progress`
  - **账号进度统一**:自评与复习结果作为单词事件写入统一进度,直接更新账号级掌握度与下次复习时间
  - **技术要点**:SRS 调度;三档结果映射为掌握度增减
- <br />
  1. 听音辨词 Listening Dash
  - **玩法**:播放单词或例句发音,从选项中选出对应单词 / 释义,限时作答
  - **学习目标**:听音辨义、发音—拼写关联
  - **核心机制**:限时、连击、生命值
  - **数据依赖**:词库 + 音频(TTS 或预生成) + `user_word_progress`
  - **账号进度统一**:听辨对错回写同一份单词进度,与拼写、阅读类游戏共享掌握度
  - **技术要点**:音频播放 + 干扰项按词向量相似度生成
- <br />
  1. 单词接龙 Word Chain
  - **玩法**:以上一个单词的末字母作为下一个单词首字母,在限定词表内限时接龙
  - **学习目标**:词汇提取速度与拼写
  - **核心机制**:计时、连锁长度、复活道具
  - **数据依赖**:词库(按首 / 末字母索引) + `user_word_progress`
  - **账号进度统一**:成功调用的单词计入熟练度,回写统一进度
  - **技术要点**:首末字母索引 + 合法性与去重校验
- <br />
  1. 字母泡泡拼词 Word Bubble Pop
  - **玩法**:打乱的字母泡泡,按正确顺序点出目标单词(乱序重排 / anagram)
  - **学习目标**:拼写与字母顺序记忆
  - **核心机制**:限时、提示、连击
  - **数据依赖**:词库 + `user_word_progress`
  - **账号进度统一**:拼对 / 拼错事件写入统一进度,与拼写蜂共享错词
  - **技术要点**:乱序生成 + 拼写校验
- <br />
  1. 同反义连线 Synonym & Antonym Match
  - **玩法**:把单词与其同义 / 反义词配对连线消除
  - **学习目标**:词义网络、近反义辨析
  - **核心机制**:配对消除、限时、连击
  - **数据依赖**:词库需含 `synonyms` / `antonyms` 字段(需补充) + `user_word_progress`
  - **账号进度统一**:配对结果回写相关单词的掌握度
  - **技术要点**:同反义关系数据 + 连线判定
- <br />
  1. 图词配对 Picture-Word Match
  - **玩法**:看图选词或词图配对,适合具象名词与场景词
  - **学习目标**:形象记忆、场景词汇
  - **核心机制**:配对、限时、连击
  - **数据依赖**:需**新增**图片资源与词—图映射 + `user_word_progress`
  - **账号进度统一**:识图对错写入统一进度
  - **技术要点**:图片 CDN + 词图映射数据
- <br />
  1. 纵横填词 Crossword Quest
  - **玩法**:按释义 / 例句提示在纵横格中填出单词
  - **学习目标**:拼写 + 词义综合运用
  - **核心机制**:关卡、提示卡、计时
  - **数据依赖**:词库 + 释义(网格由算法生成) + `user_word_progress`
  - **账号进度统一**:填词结果按单词回写统一进度
  - **技术要点**:填字网格生成算法 + 交叉校验
- <br />
  1. 词形变形闯关 Word Form Master
  - **玩法**:给定原形 / 词根,填出名词、动词、形容词、副词等派生或时态变形
  - **学习目标**:构词法与词形变化(派生 / 屈折)
  - **核心机制**:关卡、提示、连击
  - **数据依赖**:需**新增**词形变化 / 词族(word families)数据 + `user_word_progress`
  - **账号进度统一**:词族内各词形的掌握度统一归集到词条进度
  - **技术要点**:词族关系数据建模
- <br />
  1. 真题高频词挑战 High-Frequency Word Challenge
  - **玩法**:按考研真题词频从高到低闯关,优先攻克高频词
  - **学习目标**:高性价比覆盖高频考点词
  - **核心机制**:词频排序关卡、Boss 高频词包、连击
  - **数据依赖**:词库需含词频 / 考频字段 + `user_word_progress`
  - **账号进度统一**:高频词进度与其他游戏共享,避免重复背诵
  - **技术要点**:词频统计 + 关卡编排
- <br />
  1. 记忆翻牌 Memory Flip Match
  - **玩法**:翻牌配对,把「单词牌」与「释义牌」翻出并配对(记忆 / concentration)
  - **学习目标**:短时记忆 + 词形—词义映射
  - **核心机制**:步数 / 时间计分、连对奖励
  - **数据依赖**:词库 + `user_word_progress`
  - **账号进度统一**:配对成功计入熟练度,回写统一进度
  - **技术要点**:翻牌配对逻辑 + 计分

- **排行榜隐私**:默认匿名昵称,不公开真实身份信息,提供关闭排名选项
- **虚拟货币**:仅站内解锁使用,**不可提现、不可兑换现金**,避免赌博属性;抽奖类须公示概率或改为确定性掉落
- **内容安全**:PVP/组队涉及 UGC(昵称、聊天)需接内容审核过滤
- **数据合规**:游戏行为数据属个人信息,纳入隐私政策与《个人信息保护法》范畴

### 8.7 游戏化落地路线(并入总路线图)

- **阶段 1(MVP+)**:先做 2\~3 款轻量单机游戏(词义消消乐、拼写蜂、极限速算)验证留存曲线

- **阶段 3**:上线 PVP 词汇擂台、组队副本等社交玩法(需实时通信 + UGC 审核)

## 十二、🧠 AI / RAG 能力实现规格

### 12.1 向量化与检索 pipeline

- **分块**:知识点按小节、题目按「题干+解析」分块,chunk 512\~800 token、重叠约 80
- **向量**:`text-embedding-3-small`(1536 维)写入 Milvus,按 subject 分区,HNSW 索引
- **检索**:top-k 召回 + 重排;维护「query→期望知识点」评测集,持续监控召回率与命中率

### 12.2 Prompt 与引用

- 系统提示固定原则:**只依据检索到的知识点作答,给出引用编号(kp-id);无依据则明确说明,不臆造**
- 输出结构:答案 + 引用来源(知识点 id)+ 相关题目推荐

### 12.3 护栏(Guardrails)

- **幻觉控制**:强制引用来源;低置信度触发「建议教研复核」
- **数学渲染**:LaTeX 统一 `$...$` / `$$...$$`,前端 KaTeX/MathJax 渲染
- **内容安全**:输入与输出双向过内容审核(合规硬性要求)

### 12.4 成本控制

- Embedding 结果与高频问答缓存;按会员配额限流;每次调用 token 成本落 `ai_conversations`,超额降级到缓存答案或提示升级

## 十三、🔐 安全加固实施(现状 → 目标)

| 项           | 现状(不安全)                  | 目标配置                                    |
| ----------- | ------------------------ | --------------------------------------- |
| JWT 密钥      | 默认 `your-secret-key`     | 32+ 字节随机密钥,注入自 Secrets,缺失即拒绝启动          |
| MinIO 凭据    | `minioadmin/minioadmin`  | 强随机凭据,走环境变量 / Secrets                   |
| Postgres 口令 | `postgres/postgres`      | 强口令 + 最小权限账号                            |
| 内部端口        | 5432/6379/19530 映射宿主机    | 仅 `smartlearn-network` 内网,不 `ports` 到主机 |
| Redis       | 无密码 + 端口外露               | 设 `REDIS_PASSWORD`  • 内网                |
| 默认环境        | `ENVIRONMENT=production` | 默认 development;生产显式注入                   |
| TLS         | 无证书,443 不可用              | 签发证书 + 强制 HTTPS/HSTS                    |
| CORS        | 过宽                       | 白名单真实域名                                 |
| 密钥管理        | 明文入 `.env`               | Vault / 云 KMS / Docker secrets          |
