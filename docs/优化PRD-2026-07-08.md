## 0. 本轮核查结论（代码抽样核对真实完成度）

第一轮解决了部署阻断与安全底线（G1-G5），但**功能层面仍大面积空心**。本轮抽样核对 7 大代码区域，核心发现：

| 区域 | 真实状态 | 核心缺口 |
|---|---|---|
| **student-web** | App.tsx 仅 5 个路由，**全是游戏**（GameHall/WordGame/MathGame/CrossSubjectGame/GameResult） | **无登录页、无词汇学习页、无题库练习页、无真题模拟页、无 AI 辅导页、无个人中心**；client.ts 无 axios 鉴权拦截器，硬编码 `student_001` |
| **mobile** | 路由/页面齐全（Auth/Home/Learn/Word/Knowledge/Profile） | RootNavigator **从未调用 checkAuth()** → 重启登录态丢失；CalculateQuestion `onAnswer('')` 提交空串→计算题恒错；LearnScreen 题型筛选未传参+首题 timeSpent 天文数字；apiClient BASE_URL 硬编码；PlanScreen 425 行死代码 |
| **admin** | 路由完整（10 个功能模块）+ ProtectedRoute + axios 拦截器带 token | 5 处裸 `fetch()` 不带 Authorization（word/user/question/statistics 导出 + WorkbookDetail）；tsconfig strict 下 noUnusedLocals/noUnusedParameters 可能致 tsc 失败 |
| **api 后端** | auth.py 完整（register/login/refresh/me/logout/change-password）；vocab.py 有 SRS 算法；questions.py 有判分+错题本 | vocab/questions/games **全用内联 raw `Table()`** 非 ORM（P1-6 残留）；questions /recommend 实为 `ORDER BY random()` 非基于掌握度；games.get_game_detail 引用不存在的字段 |
| **ai-engine** | word_games_router 全接口存在；chat 有 require_auth+503；media 有 SSRF 防护 | **word_games_service 词库仅 15 个硬编码单词**（apple/book/cat…）；session+leaderboard **进程内内存**（不持久/跨 worker 丢失）；chat/media LLM 调用**仍同步阻塞**（P1-1 未修） |
| **data/** | kaoyan-words.json 220 条；games-config 25 款；真题/习题册仅样例 | 词汇 220 条远低于 2000+ 目标；游戏词库与 data/ 词汇完全脱节 |
| **CI/测试** | 仅 ai-engine/tests/conftest.py | **无 GitHub Actions CI**；QA 隔离单测未固化进仓库 |

**一句话结论**：第一轮让系统能跑起来，但跑起来后学生端只有游戏可玩、词库只有 15 个单词、后端用裸 SQL 而非 ORM、AI 调用仍同步阻塞。本轮要把这些"能跑但空心"的全部填实。

## 2. 用户故事

- **US1（学生-Web）**：作为学生，我希望在 student-web 上用账号密码登录，然后能背单词（翻卡+复习提醒）、刷题库、做真题、问 AI 辅导、看个人进度，这样我能在一个平台完成考研学习全流程。
- **US2（学生-移动端）**：作为学生，我希望 App 重启后不用重新登录，计算题能正常判分，练习时能按题型筛选，这样移动端学习体验不掉链子。
- **US3（学生-游戏）**：作为学生，我希望单词游戏有足够多的词汇（不是反复练 apple/cat），游戏结束后成绩和排行榜能持久保存，这样游戏有挑战性和长期激励。
- **US4（管理员）**：作为管理员，我希望管理后台能正常导出数据（不因缺鉴权 401），TypeScript 编译无报错，这样我能稳定运维后台。
- **US5（运维/安全）**：作为运维，我希望 AI 引擎的 LLM/媒体调用不阻塞事件循环（高并发不卡死），供应商故障显式报错而非假答案，这样服务稳定可观测。
- **US6（后续开发者）**：作为新接手开发者，我希望后端业务路由用 ORM 而非裸 SQL Table（可维护），CI 自动跑测试（防回归），这样我能安全迭代。
- **US7（内容运营）**：作为运营，我希望词汇库有 2000+ 真实考研词汇且格式对齐数据库，游戏题目从词库动态生成，这样无需手工维护游戏题量。
